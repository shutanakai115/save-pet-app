# ADR-001: 技術スタック選定

## ステータス

採用

## コンテキスト

つもり貯金アプリのMVPを開発するにあたり、技術スタックを決定する必要がある。
このプロジェクトは Next.js と Go のキャッチアップも兼ねている。

## 決定

以下の技術スタックを採用する：

| レイヤー | 技術 | 理由 |
|---------|------|------|
| フロントエンド | Next.js | React ベースでSSR/SSG対応 |
| UIライブラリ | Ark UI + Panda CSS | ヘッドレスUIでカスタマイズ性が高い、既存セットアップ済み |
| 状態管理 | TanStack DB | オプティミスティック更新、Live Queries によるリアルタイム性 |
| スキーマ検証 | Valibot | 軽量、Standard Schema互換、TanStack DBとの統合 |
| 認証 | Clerk | 実装コストが低く、PWA対応も可能 |
| バックエンド | Go（標準ライブラリ） | 学習目的、シンプルで高パフォーマンス |
| DBアクセス | sqlc | SQLから型安全なコード生成 |
| データベース | PostgreSQL | 信頼性が高く、本格的な構成 |
| 形態 | PWA | 「その瞬間」に入力する体験のため、ネイティブアプリに近いUXが必要 |

## 結果

### 良い点

- モダンな技術スタックで実務に活かせる
- PWA対応でモバイルでの使い勝手が良い
- TanStack DB によりオフライン対応や即時反映が可能

### 注意点

- TanStack DB は Beta 版（v0）のため、破壊的変更の可能性あり
- 技術スタックが多く、学習コストがある
