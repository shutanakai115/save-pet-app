# ADR-003: データベースとして PostgreSQL を採用

## ステータス

採用

## コンテキスト

アプリのデータ永続化のためデータベースが必要。以下の選択肢を検討した：

1. **PostgreSQL** - 本格的なRDBMS
2. **SQLite** - シンプル、ローカル開発向け
3. **Supabase** - PostgreSQL + リアルタイム機能 + 認証

## 決定

**PostgreSQL** を採用する。

## 理由

### PostgreSQL を選んだ理由

- **信頼性と実績**: 本番環境での実績が豊富
- **Go との相性**: `database/sql` や `sqlx` で容易に接続可能
- **学習価値**: 実務で広く使われており、学習しておく価値がある
- **将来の拡張性**: 複雑なクエリやインデックスにも対応可能

### 他の選択肢を見送った理由

| 選択肢 | 見送った理由 |
|--------|-------------|
| SQLite | 本番デプロイ時の制約、学習目的としては物足りない |
| Supabase | 認証はClerkを使うため、Supabase全体は不要 |

## 結果

- Go + PostgreSQL の組み合わせを学習できる
- Railway等へのデプロイも容易
- 将来的なスケーリングにも対応可能
