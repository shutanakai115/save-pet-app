# つもり貯金アプリ 要件定義

## アプリ概要

使ったつもり貯金でキャラクターを育成するアプリ

> ※キャラクター育成要素は後回しにし、まずはシンプルなつもり貯金アプリとして実装

## ターゲット・利用シーン

| 項目 | 内容 |
|------|------|
| **Who** | お金をつい無駄遣いしてしまいがちな若者 |
| **When** | 無駄な買い物をしてしまいそうになっているその瞬間 |
| **Where** | 買い物全般（コンビニ、ネットショッピングなど） |
| **What** | シンプルなつもり貯金トラッカー |
| **How** | 使ったつもりの金額を入力することで貯金額を可視化し、無駄遣いを止める |
| **Why** | 目標金額の貯金のため |

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロントエンド | Next.js |
| UIライブラリ | Ark UI + Panda CSS |

> **目的**: Next.jsとGoのキャッチアップも兼ねる

## カスタマージャーニー

### シーン：コンビニでお菓子を買おうとしている瞬間

1. **認識** - 「あ、このお菓子美味しそう...でも別に必要じゃないな」
2. **決断** - 「我慢しよう、貯金に回そう」
3. **記録** - スマホを取り出してPWAを開く → 金額入力(例:300円)
4. **フィードバック** - 「貯金額が増えた！」「目標まであと○円」の表示
5. **継続** - 定期的に振り返りでモチベーション維持

### その他の接点

- **目標設定時**: 初回起動 or 定期的な見直し
- **振り返り時**: 週末や月末に貯金額・履歴を確認
- **達成時**: 目標金額到達の喜び、次の目標設定

## ユースケース（MVP）

### 必須機能

1. **ユーザー登録・ログイン**
2. **目標を設定する**（目標名 + 目標金額）
3. **「使ったつもり」金額を記録する**（名称 + カテゴリ + 金額）
4. **貯金状況を確認する**（現在の貯金額、目標までの残額、達成率）
5. **目標の進捗を確認する**（目標名、円形ゲージ、残額）
6. **入力履歴を確認・編集・削除する**

### 主要な画面構成

- ログイン/登録画面
- ホーム画面（貯金額・達成率・CTA・最近の記録）
- クイック入力モーダル（ステップ式: 名称/カテゴリ → 金額 → 完了）
- 履歴一覧画面（月別グループ化）
- 目標画面（目標名・金額・進捗ゲージ）
- 設定画面（アカウント設定）

## 重要なUX要件

- **「その瞬間」に入力する**体験を重視
- ステップ式モーダルで入力に集中できるUX
- PWA対応（オフライン動作、ホーム画面追加）
- 豚キャラクター（Piggy Bank）を軸にした楽しいビジュアル

## 将来的な拡張要素（後回し）
