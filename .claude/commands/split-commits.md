# コミット分割

## 目的
- 変更を読みやすい単位に分け、レビューしやすくする
- このプロジェクトのコミット規約（gitmoji + 日本語1行、bodyなし）を守る

## 手順

### Step 1: 変更の全体把握
- `git status`
- `git diff --stat`

必要なら `git diff` で内容を確認し、意図の異なる変更を見分ける。

### Step 2: 論理グループ化
- 機能追加
- リファクタ
- デザイン調整
- ルール/ドキュメント

ファイル単位で分けにくい場合は、先に最小単位へ実装を整理してからコミットする。

### Step 3: メッセージ案の作成
各グループごとに、次を満たす1行メッセージを作る。
- 先頭は gitmoji
- 日本語で端的に「なぜ/何をしたか」を表す
- body は付けない

例:
- `✨ 目標作成フローを追加`
- `♻️ BottomNavのactive判定を整理`
- `📝 RSC境界ルールを追加`

### Step 4: グループごとにコミット
1グループずつ以下を繰り返す。
1. `git add <files...>`
2. `git diff --cached --stat` でステージ内容を確認
3. `git commit -m "<gitmoji + 日本語1行>"`

### Step 5: 結果確認
- `git log --oneline -n <件数>`
- `git status`

意図しない混在があれば、次の作業に入る前に整理する。

## gitmoji 早見表
- `✨` 新機能
- `🐛` バグ修正
- `♻️` リファクタ
- `🎨` UI/デザイン改善
- `⚡️` パフォーマンス改善
- `✅` テスト追加・更新
- `📝` ドキュメント・ルール更新
- `🔧` 設定変更

## 注意点
- 1コミット1目的を徹底する
- push後の `--amend` は原則しない
- 大きな差分は先に分割方針を決めてから `git add` する
